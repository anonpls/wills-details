# Модуль добавления товаров в каталог

## Описание
Модуль позволяет администраторам добавлять товары в каталог прямо из веб-интерфейса.

## Установка администратора

Чтобы пользователь мог добавлять товары, ему нужны права администратора.

### Способ 1: Через SQL (SQLite)

```bash
# Подключитесь к базе данных
sqlite3 cabinet.sqlite3

# Обновите существующего пользователя на администратора
UPDATE users SET is_admin = 1 WHERE phone = '+7'; # Замените на нужный номер телефона

# Или создайте нового администратора
INSERT INTO users (name, phone, email, password_hash, points, is_admin, created_at) 
VALUES ('Admin', '+7999999999', 'admin@example.com', 'pbkdf2:sha256$600000$...', 500, 1, datetime('now'));
# Примечание: password_hash нужно сгенерировать с помощью generate_password_hash() из werkzeug.security
```

### Способ 2: Программно

```python
from werkzeug.security import generate_password_hash

# В Python интерпретаторе или скрипте:
hash = generate_password_hash('your_password')
print(hash)

# Затем вставить в БД через SQL выше
```

## Как использовать

1. **Залогиниться** с аккаунтом администратора на сайте
2. Перейти на страницу **"Каталог"** ([http://localhost:5000/catalog](http://localhost:5000/catalog))
3. В сетке товаров внизу появится кнопка **"Добавить товар"** (пунктирный прямоугольник с крестиком)
4. Нажать на кнопку - откроется модальное окно
5. Заполнить форму:
   - **Название товара** - обязательное, макс 200 символов
   - **Категория** - обязательная, например "ВИБРОИЗОЛЯЦИЯ"
   - **Цена** - обязательная, целое число рублей
   - **Описание** - обязательное, макс 1000 символов
   - **URL изображения** - необязательное, например "/img/part_1.png"
   - **Бейджи** - необязательное, перечислить через запятую
6. Нажать "Добавить товар"
7. Товар появится в каталоге и будет сохранён в БД

## Структура товара в БД

Таблица `products`:
- `id` - PRIMARY KEY, автоинкремент
- `name` - название товара
- `category` - категория товара
- `price` - цена в рублях
- `description` - описание товара
- `image_url` - URL изображения
- `badges` - JSON массив бейджей
- `created_at` - время создания (ISO 8601)
- `created_by` - ID пользователя, который создал товар

## API Endpoints

### GET /api/products
Получить все товары из каталога

**Ответ:**
```json
{
  "products": [
    {
      "id": 101,
      "name": "Legenda 4",
      "cat": "ВИБРОИЗОЛЯЦИЯ",
      "price": 650,
      "desc": "Премиальная виброизоляция...",
      "img": "/img/part_1.png",
      "badges": ["Убирает гул", "Толщина 4мм"]
    }
  ]
}
```

### POST /api/products
Добавить новый товар (только для администраторов)

**Требует:**
- Авторизация
- Права администратора (is_admin = 1)

**Тело запроса:**
```json
{
  "name": "Новый товар",
  "category": "КАТЕГОРИЯ",
  "price": 1000,
  "description": "Описание товара",
  "image_url": "/img/new_product.png",
  "badges": ["Бейдж 1", "Бейдж 2"]
}
```

**Ответы:**
- `200 OK` - товар добавлен успешно
- `400 Bad Request` - некорректные данные
- `401 Unauthorized` - не авторизован
- `403 Forbidden` - нет прав администратора
- `500 Internal Server Error` - ошибка сервера

## Обновления базы данных

При первом запуске приложения автоматически:
1. Если таблица `products` не существует - она создаётся
2. Если поле `is_admin` отсутствует в таблице `users` - схема не обновляется автоматически

### Миграция старой БД

Если у вас уже есть БД без поля `is_admin`, выполните:

```bash
sqlite3 cabinet.sqlite3

# Добавить поле is_admin к существующим пользователям
ALTER TABLE users ADD COLUMN is_admin INTEGER NOT NULL DEFAULT 0;

# Сделать одного пользователя администратором
UPDATE users SET is_admin = 1 WHERE id = 1; # Замените ID на нужный
```

## Интеграция с фронтенд

После добавления товара:
1. Форма очищается
2. Товары перезагружаются с сервера
3. Сетка каталога обновляется
4. Показывается уведомление об успехе

## Безопасность

- ✅ Проверка прав администратора на сервере
- ✅ Валидация всех данных на сервере
- ✅ Ограничение на длину строк
- ✅ Защита от SQL-инъекций (используется parameterized queries)
- ✅ Требуется авторизация для добавления товаров
